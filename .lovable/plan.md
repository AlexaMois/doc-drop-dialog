

## Улучшение проверки дубликатов: нечёткий поиск в реальном времени

### Что изменится

Сейчас проверка на дубликаты запускается только при нажатии "Отправить". Новая реализация будет:
- Проверять название в реальном времени при вводе (с небольшой задержкой, чтобы не перегружать сервер)
- Использовать нечёткий поиск (поиск по подстроке + расстояние Левенштейна) для обнаружения похожих названий с опечатками
- Показывать результаты прямо под полем "Название документа" ещё до нажатия кнопки "Отправить"
- Разделять результаты на "точные совпадения" (жёсткое предупреждение) и "похожие документы" (мягкое предупреждение)

### Пользовательский опыт

**Точное совпадение** (красная карточка под полем):
> "Документ с таким названием уже есть в базе. Загружен: Андреев Д.В., 14.02.2026. Проверьте, нужна ли новая версия?"

**Похожие названия** (жёлтая карточка под полем):
> "Найдены похожие документы (3 шт.). Убедитесь, что загружаете новый документ, а не дубликат."

Пользователь видит предупреждение заранее, но может продолжить заполнение и отправку формы.

### Технические детали

#### 1. Edge Function -- улучшение `check-duplicate` (`supabase/functions/bpium-api/index.ts`)

Добавить нечёткий поиск с расстоянием Левенштейна:
- Реализовать функцию `levenshteinDistance` прямо в edge function
- Поиск по подстроке: если введённый текст содержится в названии или наоборот
- Порог похожести: расстояние Левенштейна <= 30% от длины строки
- Возвращать два массива: `exactMatches` (точные) и `similarMatches` (похожие)
- Для каждого совпадения возвращать: название, ФИО, дата, статус (из поля 12)

#### 2. Новый хук `useDuplicateCheck` (`src/hooks/useDuplicateCheck.ts`)

- Принимает строку `documentName`
- Debounce 800ms -- отправляет запрос только после паузы в наборе текста
- Минимальная длина для проверки: 5 символов
- Возвращает: `{ exactMatches, similarMatches, isChecking }`

#### 3. Компонент предупреждения под полем (`src/components/DuplicateInlineWarning.tsx`)

- Показывается прямо под полем "Название документа" (вместо диалога)
- Два варианта оформления:
  - Красная рамка -- точное совпадение (с деталями: кто, когда, статус)
  - Жёлтая рамка -- похожие документы (список названий)
- Индикатор загрузки (спиннер) во время проверки

#### 4. Обновление формы (`src/components/DocumentForm.tsx`)

- Подключить `useDuplicateCheck` с привязкой к `documentName`
- Вставить `DuplicateInlineWarning` после поля ввода названия
- Убрать проверку дубликатов из `onSubmit` -- она больше не нужна при отправке, т.к. пользователь уже предупреждён
- Оставить `DuplicateWarningDialog` как финальную защиту для точных совпадений при нажатии "Отправить"

#### 5. Обновление `useBpiumCatalogs.ts`

- Обновить интерфейс `DuplicateResult` для поддержки `exactMatches` и `similarMatches`
- Обновить функцию `checkDocumentDuplicate` для нового формата ответа
