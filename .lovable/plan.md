

## Проблема

Второй документ Конопелькина **не дошёл до сервера** — в логах backend есть только один `submit-document`. После нажатия «Добавить ещё документ» форма сбрасывает квиз (`quizPassed = false`), что блокирует кнопку отправки. Пользователь, вероятно, не заметил, что кнопка неактивна, и решил, что документ отправлен.

## Что нужно исправить

### 1. Не сбрасывать квиз при повторной отправке
**Файл:** `src/components/DocumentForm.tsx`, функция `handleReset` (строка 218)

Удалить `setQuizPassed(false)` — если пользователь уже прошёл квиз в текущей сессии, повторно проходить его не нужно.

### 2. Сделать блокировку кнопки по всем полям, не только по квизу
**Файл:** `src/components/DocumentForm.tsx`, строка 449

Сейчас: `disabled={isSubmitting || !quizPassed}` — кнопка блокируется только по квизу и отправке.

Нужно: добавить проверку всех обязательных полей в `disabled`, чтобы кнопка визуально отражала реальное состояние формы.

### 3. Показывать статус отправки после сабмита
**Файл:** `src/components/DocumentForm.tsx`

Добавить toast-уведомление при успешной отправке (сейчас есть только при ошибке). Пользователь должен видеть явное подтверждение «Документ отправлен».

### 4. Логировать попытки отправки на клиенте
**Файл:** `src/components/DocumentForm.tsx`

Добавить `console.log` в начало `onSubmit` до проверок, чтобы при будущей отладке было видно, дошёл ли пользователь до функции отправки, и на каком этапе произошла блокировка (валидация каталогов, дубликаты, и т.д.).

## Итог

Основная причина — сброс квиза при повторной отправке. Кнопка становится неактивной, но пользователь этого не замечает. Исправление минимальное: убрать одну строку + усилить обратную связь.
