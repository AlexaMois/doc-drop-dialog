# Техническая документация: Портал загрузки документов АТС

## 1. Обзор проекта

**Назначение:** Веб-приложение для загрузки и классификации документов в систему Bpium (BPM-платформа) компании ООО «АкТрансСервис» (АТС) — транспортного предприятия, обслуживающего буровые и промышленные объекты.

**Стек технологий:**

| Слой | Технология |
|------|-----------|
| Фреймворк | React 18 + TypeScript 5 |
| Сборщик | Vite 5 |
| Стилизация | Tailwind CSS 3 + shadcn/ui |
| Формы | React Hook Form + Zod |
| Состояние сервера | TanStack React Query 5 |
| Маршрутизация | React Router 6 |
| Backend | Supabase (Edge Functions, Storage) |
| AI | Lovable AI Gateway (Gemini 2.5 Flash) |
| Внешний API | Bpium REST API (Basic Auth) |

### Поток отправки документа

```
1. Пользователь заполняет форму
2. Файл загружается в Supabase Storage → публичный URL
3. Данные формы + URL файла отправляются в Edge Function bpium-api
4. Edge Function формирует запрос к Bpium REST API
5. Bpium создаёт запись в каталоге «Документы» (ID 56)
6. Ответ возвращается пользователю
```

## 4. Компоненты

### 4.1 DocumentForm

**Файл:** `src/components/DocumentForm.tsx`  
**Назначение:** Основная форма загрузки документа. Управляет всем жизненным циклом отправки.

**Схема валидации (Zod):**

| Поле | Тип | Валидация |
|------|-----|-----------|
| `documentName` | `string` | Обязательное, min 1 символ |
| `file` | `File` | Обязательное, проверка наличия `name` и `size` |
| `responsiblePerson` | `string` | Обязательное, min 1 символ |
| `sources` | `string[]` | Min 1 элемент |
| `directions` | `string[]` | Min 1 элемент |
| `roles` | `string[]` | Min 1 элемент |
| `projects` | `string[]` | Min 1 элемент |
| `tags` | `string[]` | Min 1 элемент |

**Логика отправки (`onSubmit`):**

1. Валидация значений справочников (проверка что ID существуют в каталогах)
2. Сохранение ФИО при первой отправке (блокировка навсегда)
3. Проверка точных дубликатов → диалог подтверждения
4. Загрузка файла в Supabase Storage (`uploadDocumentFile`)
5. Отправка данных в Bpium (`submitDocumentToBpium`)
6. Экран успеха: «ТЫ МОЛОДЕЦ!»

**Состояния компонента:**

- `isSubmitting` — процесс отправки
- `isSubmitted` — успешная отправка (показ экрана успеха)
- `quizPassed` — пройдена ли мини-викторина
- `duplicateDialogOpen` — открыт ли диалог дубликатов

**Файл:** `src/hooks/useBpiumCatalogs.ts`  
**Назначение:** Загрузка всех справочников Bpium одним запросом.

```typescript
const catalogs = useAllCatalogs();
// catalogs.sources.data → CatalogOption[]
// catalogs.directions.data → CatalogOption[]
// catalogs.roles.data → CatalogOption[]
// catalogs.projects.data → CatalogOption[]
// catalogs.isLoading → boolean
// catalogs.isError → boolean
```

- Используется `react-query` с `staleTime: 5 мин`, `gcTime: 10 мин`
- Вызывает Edge Function `bpium-api?action=get-catalogs`

### 5.2 useDuplicateCheck

**Файл:** `src/hooks/useDuplicateCheck.ts`  
**Назначение:** Проверка дубликатов по названию документа в реальном времени.

```typescript
const { exactMatches, similarMatches, isChecking } = useDuplicateCheck(documentName);
```

- **Debounce:** 800ms
- **Минимальная длина:** 5 символов для запуска проверки
- **AbortController:** отмена предыдущего запроса при новом вводе
- Вызывает Edge Function `bpium-api?action=check-duplicate`

### 5.3 useAiTagSuggestions

**Файл:** `src/hooks/useAiTagSuggestions.ts`  
**Назначение:** AI-генерация тегов на основе заполненных полей формы.

```typescript
const { suggestedTags, isLoading, error } = useAiTagSuggestions({
  documentName, fileName, sources, directions, roles, projects,
});
```

**Входные данные:** Принимает **labels** (не ID) справочников для лучшего контекста AI.

- **Debounce:** 1000ms
- **Триггер:** Любое изменение в полях (название, файл, справочники)
- **Минимум данных:** Название >2 символов ИЛИ хотя бы один справочник выбран
- **AbortController:** отмена при новом запросе
- Обработка HTTP 429 (rate limit) и 402 (оплата)

---

### 5.4 useResponsiblePerson

**Файл:** `src/hooks/useResponsiblePerson.ts`  
**Назначение:** Управление ФИО ответственного с однократной фиксацией.

```typescript
const { name, isLocked, isLoading, saveName, updateTempName } = useResponsiblePerson();
```

**Логика:**
1. При загрузке — проверка `localStorage["ats_responsible_person"]`
2. Если найдено → `isLocked = true`, поле заблокировано
3. При первой отправке → `saveName()` записывает в localStorage навсегда
4. `updateTempName()` — обновление state без сохранения (для контролируемого input)

**Ключ localStorage:** `ats_responsible_person`

## 6. Edge Functions

### 6.1 bpium-api

**Файл:** `supabase/functions/bpium-api/index.ts`  
**Назначение:** Прокси для всех операций с Bpium REST API.

**Действия (query param `action`):**

| Action | Метод | Описание |
|--------|-------|----------|
| `get-catalogs` | GET | Загрузка всех справочников (directions, roles, projects, sources) |
| `get-catalog-structure` | GET | Структура конкретного каталога (для отладки) |
| `check-duplicate` | POST | Проверка дубликатов по названию документа |
| `submit-document` | POST | Создание записи в каталоге документов |

**Аутентификация Bpium:**
```
Authorization: Basic base64(BPIUM_LOGIN:BPIUM_PASSWORD)
```

**Нормализация домена:**
- Удаляет trailing slashes
- Добавляет `https://` если отсутствует

**Алгоритм проверки дубликатов (`check-duplicate`):**

1. Загрузка всех записей каталога документов (ID 56)
2. Нормализация: `trim().toLowerCase()`
3. **Точное совпадение:** `title === input`
4. **Подстрока:** `title.includes(input) || input.includes(title)` (при длине ≥5)
5. **Расстояние Левенштейна:** `distance ≤ 30% от max(len(a), len(b))` (при длине ≥5)

**Формат данных для Bpium (`submit-document`):**

| Тип поля | Формат в Bpium |
|----------|----------------|
| Текст | `"значение"` |
| Связанный объект (multi) | `[{catalogId: "55", recordId: "123"}, ...]` |
| Связанный объект (single) | `[{catalogId: "59", recordId: "456"}]` (массив из 1) |
| Файл | `[{src: "https://...", title: "file.pdf"}]` |
| Контакт/сайт | `[{contact: "https://...", comment: ""}]` |
| Статус (dropdown) | `["1"]` (1=Черновик) |
| Дата | ISO строка |
| Теги | `"тег1, тег2, тег3"` (строка через запятую) |

### 6.2 suggest-tags

**Файл:** `supabase/functions/suggest-tags/index.ts`  
**Назначение:** AI-генерация тегов через Lovable AI Gateway.

**Модель:** `google/gemini-2.5-flash`  
**Температура:** 0.3  
**Endpoint:** `https://ai.gateway.lovable.dev/v1/chat/completions`  
**Авторизация:** `Bearer LOVABLE_API_KEY` (секрет, устанавливается автоматически)

**System Prompt включает:**
- Описание компании АТС
- Глоссарий 30+ аббревиатур (БДД, ВЧНГ, ПБОТОС, СИЗ и т.д.)
- 6 категорий тегов:
  1. Вид документа (Инструкция, Приказ, Регламент…)
  2. Тематика (БДД, ОТ, Пож.без…)
  3. Заказчик/проект (ВЧНГ, ГПНЗ, СН…)
  4. Подразделение (ВЧПУ, ДПУ, Автоколонна…)
  5. Должность/роль (Водитель, Механик, ИТР…)
  6. Процесс (Ознакомление, Контроль, Аудит…)

**Правила генерации:**
- 7–10 тегов
- На русском, 1–3 слова
- Минимум 2 категории
- Формат ответа: JSON-массив строк

**Парсинг ответа:**
- Regex поиск JSON-массива в тексте: `/\[[\s\S]*?\]/`
- Фильтрация пустых строк
- Лимит 10 тегов

## 8. Хранилище файлов

**Бакет:** `documents` (Supabase Storage)  
**Путь:** `uploads/{timestamp}-{randomId}.{ext}`

### Процесс загрузки

1. Генерация уникального имени: `{Date.now()}-{random8chars}.{ext}`
2. Загрузка через `supabase.storage.from('documents').upload()`
3. Получение публичного URL через `getPublicUrl()`
4. URL передаётся в Bpium как `[{src: url, title: originalName}]`

**Файл:** `src/lib/storage.ts`

```typescript
// Загрузка
const url = await uploadDocumentFile(file);

// Удаление (по URL)
await deleteDocumentFile(fileUrl);
```

## 10. Бизнес-логика

### Валидация формы

1. **Zod-схема:** Все поля обязательны (кроме `websiteUrl`)
2. **Валидация справочников:** Перед отправкой проверяется, что все выбранные ID существуют в загруженных каталогах
3. **Размер файла:** Максимум 50 МБ (проверка на клиенте)

### Проверка дубликатов

Двухуровневая система:

| Этап | Когда | Поведение |
|------|-------|-----------|
| Inline (при вводе) | Debounce 800ms, от 5 символов | Предупреждение под полем |
| Диалог (при отправке) | При наличии точных совпадений | Модальное окно с подтверждением |

Пользователь может игнорировать дубликаты и отправить документ повторно.

### Блокировка ФИО

- ФИО вводится один раз при первой отправке
- Сохраняется в `localStorage["ats_responsible_person"]`
- При повторном визите — поле заблокировано (`disabled + Lock icon`)
- Невозможно изменить без очистки localStorage

### Quiz-гейт

- Кнопка «Отправить в портал» заблокирована (`disabled`) до правильного ответа
- Случайный вопрос из 13 при каждой загрузке формы
- При сбросе формы — quiz сбрасывается, нужно отвечать снова

### Статус документа

Все документы создаются со статусом **«Черновик»** (ID `1`). Дальнейшая обработка (проверка, утверждение) происходит в Bpium.

## 11. Зависимости от внешних сервисов

| Сервис | Использование | Fallback |
|--------|--------------|----------|
| Bpium API | Справочники, дубликаты, создание записей | Экран ошибки с кнопкой «Попробовать снова» |
| Supabase Storage | Хранение файлов | Ошибка toast |
| Lovable AI Gateway | Генерация тегов | Пустой массив, ручной ввод тегов |

## 12. Развёртывание

- **Frontend:** Автоматический деплой через Lovable
- **Edge Functions:** Автоматический деплой при изменении файлов в `supabase/functions/`
- **Storage:** Бакет `documents` должен существовать с публичным доступом
- **Секреты:** Настраиваются через Lovable Cloud
